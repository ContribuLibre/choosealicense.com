Sem licença

Quando você cria uma obra criativa (o que inclui código), a obra fica, por padrão, sob direitos autorais exclusivos. A menos que você inclua uma licença que especifique o contrário, ninguém mais pode copiar, distribuir ou modificar sua obra sem correr o risco de remoções, intimidações ou litígios. Uma vez que a obra passa a ter outras pessoas que contribuem (cada uma titular de direitos autorais), “ninguém” começa a incluir você.

Mesmo na ausência de um arquivo de licença, você pode conceder alguns direitos nos casos em que publica seu código-fonte em um site que exige a aceitação de termos de serviço. Por exemplo, se você publica seu código-fonte em um repositório público no GitHub, você aceitou os Termos de Serviço, pelos quais permite que outras pessoas vejam e façam fork do seu repositório. Outras pessoas podem não precisar da sua permissão se limitações e exceções aos direitos autorais se aplicarem à situação específica delas. Nem os termos do site nem as limitações de direitos autorais específicas de uma jurisdição são suficientes para os tipos de colaboração que as pessoas geralmente buscam em um hospedeiro público de código, como a experimentação, a modificação e o compartilhamento promovidos por uma licença de código aberto.

Você não precisa fazer nada para não oferecer uma licença. No entanto, você pode querer adicionar um aviso de direitos autorais e uma declaração de que não está oferecendo nenhuma licença em um lugar de destaque (por exemplo, o README do seu projeto), para que as pessoas usuárias não presumam que você cometeu um descuido. Se você for aceitar contribuições de outras pessoas ao seu projeto sem licença, pode querer estudar, com seu advogado, a adição de um acordo de contribuição ao seu projeto, para que você mantenha a permissão de direitos autorais de quem contribui, mesmo que você não conceda o mesmo.

Proibir o uso do seu código talvez não seja o que você pretende com “sem licença”. Uma licença de código aberto permite a reutilização do seu código mantendo os direitos autorais. Se o seu objetivo é abrir mão totalmente das restrições de direitos autorais, experimente uma dedicação ao domínio público.

Para quem usa

Se você encontrar um software que não tem licença, isso geralmente significa que você não tem permissão de quem criou o software para usá-lo, modificá-lo ou compartilhá-lo. Embora um hospedeiro de código como o GitHub possa permitir que você veja e faça fork do código, isso não significa que você esteja autorizado a usar, modificar ou compartilhar o software para qualquer finalidade.

Suas opções:

  • Peça gentilmente a quem mantém o projeto para adicionar uma licença. A menos que o software contenha fortes indicações em contrário, a falta de uma licença provavelmente é um descuido. Se o software estiver hospedado em um site como o GitHub, abra uma issue solicitando uma licença e inclua um link para este site. Se você for ousado e for bastante óbvio qual licença é mais apropriada, abra um pull request para adicionar uma licença – veja “sugerir esta licença” na barra lateral da página de cada licença neste site (por exemplo, MIT).
  • Não use o software. Encontre ou crie uma alternativa que esteja sob uma licença de código aberto.
  • Negocie uma licença privada. Chame seu advogado.
