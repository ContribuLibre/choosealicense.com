开源许可证向公众授予 ✓ 许可，准许对被许可作品做一些版权或其他「知识产权」法律本可能禁止的事情。

大多数开源许可证授予的许可，都以遵守一些 ⓘ 条件为前提。

大多数开源许可证还设有一些 ✕ 限制，通常用以免除担保和责任，有时还明确将专利或商标排除在许可的授予之外。

许可

商业使用
✓ 被许可的材料及其衍生物可用于商业目的。
分发
✓ 被许可的材料可以被分发。
修改
✓ 被许可的材料可以被修改。
专利使用
✓ 本许可证明确授予来自贡献者的专利权。
私人使用
✓ 被许可的材料可以私下使用和修改。

条件

公开源代码
ⓘ 分发被许可的材料时，必须提供其源代码。
保留版权与许可声明
ⓘ 分发被许可的材料时，必须随附一份许可证和版权声明的副本。
保留声明（源代码）
ⓘ 以源代码形式分发被许可的材料时，必须随附一份许可证和版权声明的副本，但二进制形式无此要求。
网络使用即分发
ⓘ 通过网络与被许可材料交互的用户有权获得一份源代码副本。
相同许可证
ⓘ 分发被许可的材料时，修改必须以相同的许可证发布。在某些情况下，可使用相似或相关的许可证。
相同许可证（文件）
ⓘ 分发被许可的材料时，对现有文件的修改必须以相同的许可证发布。在某些情况下，可使用相似或相关的许可证。
相同许可证（库）
ⓘ 分发被许可的材料时，修改必须以相同的许可证发布。在某些情况下，可使用相似或相关的许可证，或者此条件可能不适用于将被许可材料作为库使用的作品。
声明改动
ⓘ 对被许可材料所做的改动必须予以记录。

限制

责任
✕ 本许可证包含责任限制条款。
专利使用
✕ 本许可证明确声明其不授予对贡献者专利的任何权利。
商标使用
✕ 本许可证明确声明其不授予商标权，尽管没有此类声明的许可证可能也不会授予任何隐含的商标权。
担保
✕ 本许可证明确声明其不提供任何担保。
